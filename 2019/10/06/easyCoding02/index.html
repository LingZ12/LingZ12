(二)面向对象

OOP(Object-Oriented Programming)实践了软件工程3个主要目标: 可维护性、可重用性、可扩展性

1. OOP理念

(1)Object类的三大哲学问题.

  1. 我是谁? getClass()说明本质是谁,而toString()是当前职位的名片.
  2. 我从哪里来? Object()构造方法是生产对象的基本步骤,clone()是繁殖对象的另一种方式.
  3. 我到哪里去? finalize()是在对象销毁触发的方法.

(2) clone()方法分为浅拷贝、一般拷贝和彻底深拷贝

  • 浅拷贝:只复制当前对象的所有基本数据类型,以及相应的引用变量,但没有复制引用变量指向的实际对象
  • 彻底深拷贝:在成功clone一个对象后,此对象与母对象在任何引用路径上都不存在共享的实例对象,但是引用路径越深,则越接近JVM底层对象,且发现彻底深拷贝实现的难度越大.

(3)面向对象的特性

  • 传统面向对象的三大特性: 继承、多态、封装
  • 四大特性: 继承、多态、封装、抽象

2.接口与抽象类

接口和抽象类的区别：
（1）抽象类可以有构造方法，接口中不能有构造方法。
（2）抽象类中可以有普通成员变量，接口中没有普通成员变量
（3）抽象类中可以包含静态方法，接口中不能包含静态方法
（4） 一个类可以实现多个接口，但只能继承一个抽象类。
（5）接口可以被多重实现，抽象类只能被单一继承
（6）如果抽象类实现接口，则可以把接口中方法映射到抽象类中作为抽象方法而不必实现，而在抽象类的子类中实现接口中方法

接口和抽象类的相同点：
(1) 都可以被继承
(2) 都不能被实例化
(3) 都可以包含方法声明
(4) 派生类必须实现未实现的方法

3.访问权限控制


protected:只能修饰属性和方法,不能修饰类

3.this和super

4.类的关系

类的关系主要分为五种

  • [继承] extends (is-a)
  • [实现] implements (can-do)
  • [组合] 类是成员变量 (contaiins-a)
  • [聚合] 类是成员变量 (has-a)
  • [依赖] import类 (use-a)

5.序列化(Serialization)

内存中的数据对象只有转换成二进制流才可以进行数据持久化和网络传输.将数据对象转换为二进制流的过程称为对象的序列化(Serialization).

5.1常见的三种序列化方式

1.java原生序列化
Serializable接口类是实现该类对象的序列化, 该接口没有方法, 只有标记作用; 一定要显示设置serialVersionUID属性值

  • 兼容升级, 不要修改serialVersionUID
  • 不兼容升级, 需要修改

2.Hessian 序列化

  • 支持动态类型、跨语言、基于对象传输的网络协议;
  • 自描述序列化类型(不依赖外部描述文件或接口定义)
  • 语言无关, 支持脚本语言
  • 协议简单, 比Java原生序列化高效

3.JSON 序列化

  • 将数据对象转换未JSON字符串;
  • 序列化时抛弃类型信息, 所以反序列化时只有准确提供类型才能准确的反序列化;
  • 可读性好, 方便调试

4.序列化安全

  • 序列化通过网络传输对象时, 对象中友敏感数据, 会容易成为黑客的攻击点;
  • 反序列化漏洞

5.防范序列化安全问题:

  • 使用transient 关键字,避免把此属性信息转化为二进制流.
  • 使用对称或非对称加密传输

6.方法

6.1方法签名=方法名称+参数列表

或 方法签名=方法名称+参数类型+参数个数
方法签名是JVM标识方法的唯一索引,不包括返回值、访问权限控制符、异常类型等.

6.2方法参数

f(x,y) - f(1,2)
形参:x,y
实参:1,2

6.3参数传递

分为值传递与引用传递

JAVA中参数传递都是值复制的过程
对于引用变量,复制指向对象的首地址,双方都可以通过自己的引用变量修改指向对象的相关属性

6.4可变参数

JDK1.5之后引入了可变参数,通过 “参数类型…”的方式定义
尽量少使用

使用可变参数,需要对参数进行预处理(入参保护+参数校验)

6.5 构造方法

特征

  1. 构造方法名称必须和类名相同.
  2. 构造方法是没有返回值,即使是void也不能有.
  3. 构造方法不能被继承,不能被覆写,不能被直接调用.
  4. 类定义时提供了默认的无参构造方法.
  5. 构造方法可以私有.
  • 接口中不能定义构造(抽象类可以)
  • 静态代码块优先级最高, 在父子类之前执行, 并且只运行一次;

6.6 覆写(override)

覆写:运行时的多态,动态绑定,发生在垂直方向
注意事项

  1. 访问权限不能变小.
  2. 返回类型能够向上转型成为父类的返回类型.
  3. 异常类型也要向上转型成为父类的异常.
  4. 方法名、参数类型及个数必须严格一致.

6.7重载(overloade)

重载:编译时的多态,静态绑定,发送在水平方向

方法签名= 方法名称+参数类型+参数个数

重载是指不同的函数使用相同的函数名，但是函数的参数个数或类型不同。调用的时候根据函数的方法签名来区别不同的函数。

7.泛型

  • E Element 集合中的元素
  • T the Type of object 某个类
  • K Key
  • V Value

泛型注意事项

  1. 尖括号里的每个元素都指代一种未知的类型.
  2. 尖括号的位置,必须在类名之后或方法返回值之前.
  3. 泛型定义处只具备执行Object方法的能力.
  4. 对于编译之后的字节码指令,其实没有这些花头花脑的方法签名,重复呢个说明泛型只是一种编写代码的语法检查工具.

8.基本数据类型和包装类型

包装类和基本数据类型的使用?

  1. 所有POJO类属性必须使用包装数据类型;
  2. RPC(远程过程调用)方法的返回值和参数必须使用包装;
  3. 所有的局部变量推荐使用基本数据类型;

字符串

字符串的相关类型有三种:String、StringBuffer、StringBulider

面试题

String，StringBuffer，StringBulider 三者的区别

  1. String是字符串常量(不可变)，StringBuffer 和StringBuilder是字符串变量(可变使用append)。
  2. 运行速度 StringBuilder > StringBuffer > String
  3. StringBuilder是线程不安全的,StringBuffer是线程安全的（带有synchronized关键修饰的方法）。这也就是运行速度StringBuilder 比StringBuffer 快的原因。
  4. String适合用在对少量的字符串的操作，StringBuilder适合用在单线程对可变字符串的操作，StringBuffer 适合用在多线程对可变字符串的操作。
