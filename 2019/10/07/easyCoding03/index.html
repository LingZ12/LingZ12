(三)代码风格

3.1 命名

  1. 符合语言特性
  2. 体现代码元素特征:
    Abstractxxx、 Basexxxx、xxException、xxxTest等;
    包名统一使用小写, 完整单词+点分隔符;
    枚举类成员全大写+下划线
  3. 望文生义
  4. POJO类布尔类型的变量:
    数据库使用is_xxxx不用isXxxx,但是POJO类中属性为xxxx;

3.2 风格

  1. if 不要省略中括号;
  2. 没有必要增加若干空使变量赋值等号对齐
  3. 多层嵌套不能超过3层: 可使用卫语句, 策略模式, 状态模式等实现
  4. 避免反逻辑运算符: 反例 if(!(x >= 48))
  5. 条件表达式中不允许出现复杂逻辑组合: 可以将复杂逻辑赋值给布尔变量

    3.4 注释

  6. 枚举类一定要有注释
  7. 不允许写在代码后面
